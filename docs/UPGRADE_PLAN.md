# 山海经数字化展示 — 三梯队升级规划

> 创建日期：2026-06-20 | 版本：v2.0.0

---

## 总体目标

将现有 Demo 级别的单页展示，升级为具备**生产级工程设计、沉浸式交互体验、持续迭代能力**的数字人文档案平台。

---

**现状**：26 个标记点（6 神兽 + 12 山 + 5 国 + 5 水）

**目标**：新增约 40 个标记点，使地图"充实"起来。新增内容：

- 补全南山经、西山经、北山经剩余名山（10+ 点位）

- 补充水系（8+ 点位）
- 补充更多古国（6+ 点位）

**涉及文件**：`app/src/data/mapData.ts`

#### 1.1.3 补全古籍阅读器章节（3 → 18 章）

**现状**：仅 3 章（南山经、西山经、北山经）

**目标**：扩展至完整的《山海经》18 卷

- 山经五卷：南山经(✓)、西山经(✓)、北山经(✓)、东山经(新增)、中山经(新增)

- 海内经一卷(新增)

**涉及文件**：`app/src/data/classicalTexts.ts`

### 1.2 交互升级

#### 1.2.1 WebGL 着色器首屏增强

**现状**：`VoidShader.tsx` 已存在但未在页面中使用，首屏是纯色的 `Hongmeng`。

**目标**：将 WebGL 着色器作为首屏背景层，营造"鸿蒙初辟"的宇宙感。

- 着色器绘制动态星空、流光粒子、云雾渐变
- 鼠标移动产生视差互动
- `Hongmeng` 组件叠加在着色器上方，保持半透明古籍容器

**涉及文件**：
- `app/src/sections/Hongmeng.tsx` — 整合 VoidShader 作为背景
- `app/src/sections/VoidShader.tsx` — 可能增强着色器效果

#### 1.2.2 音效系统

**现状**：完全静默，无任何音频。

**目标**：建立轻量级音效系统，增强沉浸感。

- **翻页音效**：古籍翻页时播放纸页翻动声（CSS class 触发）
- **卷轴展开音**：首屏入场时古琴/编钟音效
- **交互反馈音**：按钮点击、地图标记悬停、神兽卡片翻转等微音效
- **背景氛围音**：可选开关的古风氛围音（古琴、流水）
- **音效管理器**：单例 AudioManager，统一控制音量/静音

**涉及文件**：
- `app/src/lib/audioManager.ts` — 新建
- `app/public/audio/` — 新建音频资源目录

#### 1.2.3 动画增强

- **3D 地球性能优化**：减少经纬线密度、优化纹理加载
- **神兽列表新增 Loading 态**：悬停图片懒加载 placeholder
- **滚动叙事增强**：ScrollTrigger pin 固定段落，创造电影级卷轴效果

## 第2梯队：核心架构升级

### 2.1 PWA 离线支持

- **Service Worker**：缓存静态资源（图片、字体、CSS、JS），支持离线访问
- **Web App Manifest**：桌面/PWA 安装支持，自定义图标和启动画面
- **离线数据**：IndexedDB 缓存神兽数据、古籍文本

### 2.2 状态管理（Zustand）

- 阅读进度持久化（localStorage + Zustand persist）
- 用户偏好：简繁切换、横竖排切换、暗色模式
- 书签/收藏系统
- 全局音效开关

### 2.3 暗色模式

- 基于 CSS 变量，切换 `data-theme` 属性
- 暗色主题：墨色背景 + 金色文字 + 降低对比度保护眼睛
- 系统偏好自动检测 + 手动切换

### 2.4 性能优化

- 图片 WebP 格式 + 响应式 srcset
- 字体子集化（只加载用到的汉字）
- 构建 size budget 监控
